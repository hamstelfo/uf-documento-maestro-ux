- Criterio de Identificación de Controles UI: Un control UI se define como una abstracción funcional reutilizable que responde a un tipo de interacción del usuario. Los siguientes criterios deben aplicarse para documentar los controles de forma consistente:

  1. **Instancias idénticas:** No se documentan múltiples instancias del mismo control dentro de una pantalla. Cada tipo de control se registra una sola vez, independientemente de cuántas veces aparezca.

  2. **Variaciones de contenido:** Diferencias en texto, íconos o datos no crean nuevos controles. Un mismo tipo de control con distintos contenidos sigue siendo el mismo control funcional.

  3. **Estados visuales:** Los estados visuales (hover, pressed, disabled) no implican la creación de controles adicionales. El control se define por su funcionalidad, no por su apariencia.

  4. **Funcionalidad distintiva:** Un control se identifica cuando tiene una responsabilidad interactiva clara y puede reutilizarse en distintas pantallas.

  5. **Categorías y subcategorías de control:** Los controles se organizan en categorías generales (por ejemplo, `Botón`) y subcategorías específicas según su función (por ejemplo, `BotonConfirmarAlineacion`, `BotonMenu`). Esto permite mantener una taxonomía coherente.

  6. **Subcomponentes internos:** Los subcomponentes dentro de un control compuesto no se documentan como controles separados, a menos que tengan funcionalidad interactiva propia.

  7. **Documentación única:** Cada control se documenta una vez y se referencia desde las pantallas donde aparece.

  8. Criterio Adicional para Botones con Funcionalidades Distintas:

     1. **Funcionalidad específica:** Cada botón se considera un control independiente si su funcionalidad es diferente. Por ejemplo, `BotonConfirmarAlineacion` es una subcategoría dentro de la categoría `Botón`.

     2. **Nomenclatura descriptiva:** El nombre del control debe reflejar la acción específica que realiza el botón, asegurando claridad y precisión.

     3. **Agrupación por categoría:** Todos los botones se agrupan bajo la categoría general de `Botón`, pero cada subcategoría se define por la funcionalidad específica del botón.


  9.  Criterio de Controles y Componentes Informativos

      1. **Componentes informativos no interactivos:** Elementos visuales que no tienen interacción directa (como íconos o indicadores gráficos) se documentan como parte del control o la pantalla, pero no se clasifican como controles independientes.

      2. **Elevación a control por interactividad:** Si un componente informativo adquiere funcionalidad interactiva (por ejemplo, un indicador que abre un menú), entonces se convierte en un control y se documenta como una subcategoría del control principal.
